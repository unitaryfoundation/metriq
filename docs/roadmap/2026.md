**Note:** The items in this roadmap are not yet prioritized.

## Overview

This roadmap outlines the main directions for the second year of the Metriq project. It is intended as a public, evolving document to guide discussion, collaboration, and execution after the paper.

We break this roadmap into four areas: Credibility, Coverage, Advancement, and Adoption. The sections below describe each area in more detail.

---

## 1. Credibility

Strengthen trust in Metriq as a flexible, auditable, and FAIR-aligned benchmarking platform.

### Weighting scheme
- Revisit the current weighting and aggregation methodology used in the Metriq Score  
- Evaluate whether alternative weighting schemes would better reflect benchmark relevance and scale  

### Software stack (transpilation / compilation / QEM)
- Verify semantic equivalence and execution consistency across providers  
  → issue: #TBD  
- Better account for differences introduced by:
  - optimization passes  
  - native gate transpilation  
  - quantum error mitigation overhead  
  - barrier handling  
- Explore support for programming APIs beyond Qiskit where needed  

### Continuous running
- Run the full benchmark suite at a regular cadence (e.g., every 6 months)  
  → issue: #TBD  
- Reduce time gaps between data collection events  

### Error bars
- Report uncertainties in raw benchmark data  
- Evaluate whether uncertainty propagation to final scores is necessary, depending on observed magnitude across the suite  

### Execution provenance
- Improve metadata completeness  
  → issue: #TBD  
- Include device calibration data where available  
- Store and share raw measurement-count data when possible

  - ion-trap  
  - neutral-atom  
  - photonic

## 3. Advancement

Design and implement frontier benchmark protocols. This is the focus area that requires the most research.

### Different topologies
- Develop benchmarks that better capture the effects of hardware topology and connectivity  

### Quantum advantage / utility benchmarks
- Design and implement benchmarks targeting regimes associated with quantum advantage or utility  
  → issue: #TBD  
- Possible inspirations include:
  - Quantum Advantage Tracker  
  - quantum information supremacy protocols  
  - BlueQubit’s peaked circuits  

### Logical / fault-tolerant benchmarks
- Explore benchmarks for logical or fault-tolerant regimes  
- Clarify the overlap, if any, with quantum advantage benchmarks  

### Entanglement-based benchmarks
- Build on ideas from BSEQ  
  → issue: #TBD  
- Explore protocols based on large entangled states such as GHZ states  
- Investigate whether this could become a vendor-optimized benchmark family, in the spirit of LINPACK for supercomputers  

### Qudit system benchmarks
- Explore benchmarks for systems based on qudits rather than qubits  

---

## 4. Adoption

Grow usage, participation, and community engagement around Metriq.

### Open benchmark governance
- Define a clear process for proposing, reviewing, and integrating new benchmarks into the Metriq suite  
  → issue: #TBD  
- Potential mechanisms include:
  - working groups  
  - RFC-style proposals  
- Move away from the current ad hoc process  

### Community interaction
- Link benchmark results on the website more directly to the data repository and discussion surfaces  
  → issue: #TBD  
- Enable more engaging sharing and discussion of results  

### Focused content
- Publish content on topics such as:
  - architectural decisions behind Metriq  
  - detailed guides on how to run and contribute benchmarks  
  - retrospectives and lessons learned from cross-platform benchmarking
